> **Daily Bazar** is a comprehensive, production-ready e-commerce solution. It features a modern React storefront, a robust PostgreSQL-backed Node.js API, and a powerful React 19 Admin Dashboard. Standout features include **AI-powered semantic search via Google Gemini**, SSLCommerz payment integration, real-time analytics, and an automated 7-day return system. --- ## ✨ System Highlights * 🧠 **AI-Powered Discovery:** Semantic product search utilizing the Google Gemini API, featuring keyword fallbacks and PostgreSQL candidate pre-filtering. * 💳 **Secure Payments:** Full SSLCommerz integration for cards and mobile banking, plus Cash on Delivery (COD). Post-payment HTML invoice generation. * 📦 **Complete Order Lifecycle:** Real-time stock deduction, multi-stage tracking (Processing → Shipped → Delivered), and a secure 7-day automated return window with DB transaction rollbacks. * 🔐 **Bulletproof Authentication:** OTP-based email verification, HTTP-only JWT cookies, and secure SHA-256 hashed password resets. * 📊 **Live Analytics:** Comprehensive admin dashboard featuring Recharts, top-selling metrics, low-stock alerts, and date-range report generation. * 🖼️ **Optimized Media & Content:** Multi-image uploads via Cloudinary, Jodit rich text editing for products, and dedicated review systems. --- ## 🛠️ Architecture & Tech Stack ### 1. Storefront (Client) * **Framework:** React 18 + Vite * **Routing & State:** React Router v6, Redux Toolkit + Redux Persist * **Styling & UI:** Tailwind CSS, Lucide React, React Icons, React Toastify * **Payment & HTTP:** SSLCommerz (Redirect Flow), Axios ### 2. Admin Dashboard (Admin) * **Framework:** React 19 + Vite * **Routing & State:** React Router v7, Redux Toolkit * **Data Visualization:** Recharts * **Content Editor:** Jodit React + `@uiw/react-md-editor` ### 3. Backend API (Server) * **Runtime & Framework:** Node.js (ESM), Express.js v5 * **Database:** PostgreSQL (`pg`) * **Auth & Security:** JWT (HTTP-only cookies), Bcrypt, dotenv * **Integrations:** Google Gemini API, SSLCommerz (`sslcommerz-lts`), Cloudinary, Nodemailer, `express-fileupload` --- ## 🚀 Getting Started ### Prerequisites * Node.js v18+ * PostgreSQL database (local or hosted like Neon/Supabase) * API Keys for Cloudinary, SSLCommerz (Sandbox/Live), Nodemailer (Gmail App Password), and Google Gemini. ### 1. Database Setup The PostgreSQL tables are auto-created on server startup via the `createTables()` utility. Simply ensure your database is running and accessible via the connection string. ### 2. Environment Variables Configuration You will need to create `.env` files for all three environments. **Server (`Server/config/config.env`)** ```env # Server PORT=5000 BACKEND_URL=http://localhost:5000 FRONTEND_URL=http://localhost:5173 DASHBOARD_URL=http://localhost:5174 # Database DATABASE_URL=postgresql://user:password@host:5432/dbname # JWT JWT_SECRET=your_jwt_secret_key JWT_EXPIRE=7d COOKIE_EXPIRE=7 # Cloudinary CLOUDINARY_NAME=your_cloud_name CLOUDINARY_API_KEY=your_api_key CLOUDINARY_API_SECRET=your_api_secret # Email (Nodemailer) SMTP_HOST=smtp.gmail.com SMTP_PORT=587 SMTP_EMAIL=your_email@gmail.com SMTP_PASSWORD=your_gmail_app_password # SSLCommerz SSLCOMMERZ_STORE_ID=your_store_id SSLCOMMERZ_STORE_PASSWORD=your_store_password SSLCOMMERZ_SANDBOX=true # false for production # Gemini AI GEMINI_API_KEY=your_gemini_api_key ``` **Client (`Client/.env`) & Admin (`Admin/.env`)** ```env VITE_API_URL=http://localhost:5000/api/v1 ``` ### 3. Installation & Execution Open three separate terminals to run the full stack simultaneously: ```bash # Terminal 1: Backend Server (Runs on port 5000) cd "ECommerce(SERVER)/Server" npm install npm run dev # Terminal 2: Storefront Client (Runs on port 5173) cd "E-Commerce(CLIENT)/Client" npm install npm run dev # Terminal 3: Admin Dashboard (Runs on port 5174) cd "ECommerce(Dashboard)/ecommerce-dashboard-template" npm install npm run dev ``` --- ## 📖 Deep Dive: Features ### 🛍️ Client (Storefront) * **Authentication:** 10-minute expiry OTP email verification, robust JWT auth, avatar uploads, and secure password resets. * **Product Browsing:** Filter by price, rating (1–5), stock status, and category. Includes a zoom lightbox gallery, fashion size selectors, and dynamic stock indicators. * **AI Search:** Semantic search using Google Gemini (backend processing) with fallback to standard keyword matching. * **Cart & Checkout:** Persistent cart (survives refresh), strict stock limits, automatic calculation of 5% tax + ৳100 shipping (free over ৳5000). BD phone validation (`01XXXXXXXXX`). Post-checkout invoice rendering. * **Orders & Returns:** Tabbed status tracking. 7-day return window originating from the `paid_at` timestamp. Includes predefined return reasons and admin status tracking (Pending/Approved/Rejected). * **Verified Reviews:** Users can only review products they have successfully purchased. ### 🛡️ Admin Dashboard * **Analytics Overview:** Total/Daily/Monthly revenue comparisons, top 5 selling products, low stock alerts, Recharts monthly sales line charts, and user/subscriber counts. * **Report Generator:** Date-range picker extracting total revenue, status breakdowns, daily sales charts, and return statistics. * **Product Management:** Full CRUD. Multipart form featuring a Jodit rich text editor, Cloudinary image syncing/deletion, size variant configuration, and discount pricing. * **Order & Return Handling:** Update status from Processing to Delivered or Cancelled (automatically restoring stock/refunding). Dedicated UI to Approve/Reject returns with customer-facing admin notes. * **Communications:** Manage newsletter subscribers and auth-gated contact form messages. ### ⚙️ Backend Logic Highlights * **Payment Handlers:** SSLCommerz success webhooks are strictly validated (`val_id`) to prevent spoofing. Idempotent success handlers skip double-processing. * **Return Transactions:** Approving a return runs a rigid PostgreSQL transaction: setting the order to `Returned`, payment to `Refunded`, and restoring stock. Fails safe (rolls back) on any error. * **Notifications:** Newsletter subscribers receive automated, fire-and-forget emails when product discounts are created or updated. --- ## 🗄️ Database Schema
